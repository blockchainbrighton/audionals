Audionals JSON Standard

The Audionals JSON Standard defines a self-contained, on-chain audio file format that packages raw audio and exhaustive metadata in one immutable JSON object. It originated as a solution for encoding music as Bitcoin Ordinals where conventional formats (MP3, WAV, FLAC) were too heavyweight or restrictive. Key innovations include:

  • Comprehensive schema covering technical, contextual, preservation, administrative and user-defined metadata.
  • Base64-encoded audio stored inline, eliminating external asset dependencies.
  • UTF-8 support for global, multilingual metadata to build a world audio knowledge base.
  • Cross-platform consistency so any JSON-aware client can decode and play the file.
  • Perpetual indexing enabling simple archival, search and retrieval of sound assets on Bitcoin.

By embedding audio data directly inside JSON, the standard removes friction for dApp integration, empowers fully on-chain playback and forms the backbone for every other Audionals component.

OrdSPD – Ordinal Sample Pad Device

OrdSPD is a grid-based sampler and regenerative Bitcoin Ordinal mixer purpose-built for Audionals. It brings MPC-style beat-making into the browser while storing every pad trigger, sample reference and arrangement directly on-chain.

Feature Highlights

  • 16-pad interface for tapping and sequencing smart samples.
  • One-click actions to Save, Load, Randomise or Clear mixes.
  • Global BPM control and hotkeys (-/+, </>, m, 0).
  • Pixel-art avatars represent sample layers, matching the Bitcoin Ordinals aesthetic.
  • All compositions export as JSON Ordinals compatible with the main sequencer.

OrdSPD acts as an intuitive gateway for creators who want to craft loops before graduating to the full multitrack sequencer.

The Audionals Protocol

At its core, Audionals is a Bitcoin Ordinal standard that re-imagines music production, distribution and rights management inside a trustless, decentralised environment. Every composition is a true Web3 artefact whose ownership tree lives completely on-chain.

Cornerstone Concepts

  • Programmatic Composition — a song is an executable Merkle tree that rebuilds itself from atomic audio components when played.
  • Wallet-bound Rights — stems, samples, effects and even plugin code are each controlled by the creator’s address; the song’s JSON references those addresses as immutable proof of entitlement.
  • Zero middlemen — contracts and splits are embedded in the file structure, obviating external legal agreements.
  • Accessibility — storing only optimised JSON keeps costs low, bringing professional-grade blockchain tooling to everyday musicians.
  • Comprehensive Attribution — from vocalists to DSP script authors, every contributor is permanently credited.

Together these features unlock a transparent, collaborative and censorship-resistant musical ecosystem.

On-Chain Sequencer (BitcoinBeats BETA_XI)

The flagship BitcoinBeats sequencer is a fully-featured DAW that itself is inscribed on Bitcoin, able to generate recursive inscriptions. Musicians create, edit and publish entire multitrack projects without ever leaving the chain.

Workspace Overview

  • 16+ Channels with per-track controls: Volume (V), Pitch (P), Trim (T), Clear (C), Mute (M), Solo (S).
  • Step Sequencer Grid supporting forward (red) and reverse (green) triggers, auto-draw pattern generators and pattern shifting.
  • BPM Slider and transport for play/stop.
  • Project Ops: Save/Load presets, Copy-Paste between patterns, multi-sequence navigation, Continuous Play.
  • JSON Export: each save emits a fully portable file conforming to the Audionals JSON standard.

Technical Footprint

Implemented as a web app, the sequencer’s code, UI assets and default samples are all Ordinals. When launched, it self-hosts directly out of the blockchain, blending distribution, tooling and content into a single medium.
